# Weekly Work Summary Generator

Read all work logs from `~/.claude/work-logs/` for the target week and produce a
comprehensive, narrative summary organized by project.

## Step 1: Determine the Target Week

- **Default:** The current week, Monday through Sunday.
- If `last week (week of 2026-07-06 to 2026-07-12)` contains "last week", shift back 7 days.
- If `last week (week of 2026-07-06 to 2026-07-12)` contains a specific date, use the week containing that date.
- The week always starts on Monday and ends on Sunday.

## Step 2: Read the Logs

1. List all `.md` files in `~/.claude/work-logs/` (excluding the `summaries/` subdirectory).
2. Filter to files whose date (from the `YYYY-MM-DD` prefix in the filename) falls within
   the target week's Monday–Sunday range.
3. Read each matching file and parse both the YAML frontmatter and the markdown body.

If no logs exist for the target period, say so clearly:
> "No work logs found for the week of {date}–{date}. The work-logger skill writes entries
> during design sessions — check that it's triggering during your work."

## Step 3: Generate the Summary

Write the summary to: `~/.claude/work-logs/summaries/week-of-YYYY-MM-DD.md`
(using the Monday date of the target week).

If a summary for this week already exists, overwrite it — the latest version is
always the canonical one.

## Step 4: Present the Summary

After writing the summary file, display its full contents in the conversation so the
user can review it immediately. Mention where the file was saved.

If running as a scheduled trigger (Friday 5 PM), the summary is written to disk
at `~/.claude/work-logs/summaries/` and will be available for review.

## Handling edge cases

- **Only one session in the week:** Still generate a summary, but keep it proportionally
  brief. No need for "Patterns & Observations" with a single data point.
- **Sessions spanning midnight:** Use the date in the filename, not the time.
- **Logs with missing frontmatter fields:** Work with what's available. Don't fail
  because a tag list or milestone flag is missing — infer from the content.
- **Very large number of sessions:** Summarize at a higher level rather than listing
  every single accomplishment. The summary should be scannable in under 2 minutes.
