# 🧰 The Handoff Harness

A **design-handoff communications agent**. You point it at a finished prototype + a designer's note, and it builds one canonical *change brief*, then fans that out into a tailored draft for every downstream audience — all held behind a human-review UI before anything "sends."

Next.js 15 (App Router) · React 19 · Vercel AI SDK v6 · models via AI Gateway (`claude-opus-4.8` / `claude-sonnet-5`) · in-process, no subprocess.

---

## The loop (the actual "harness")

```
  Browser  ──POST──►  /api/run  ──►  runPipeline()  ══ NDJSON event stream ══►  Browser
 (page.tsx)                        (lib/pipeline.ts)        (live artifact cards)

 ┌──────────────────────────────────────────────────────────────────────────┐
 │  ① UNDERSTAND      ToolLoopAgent + tools → one structured Change Brief     │
 │     baseline:  codebase-diff  ▸  url-diff (screenshot+HTML)  ▸  inferred   │
 │     tools:  fetchPrototype · readReference · inspectPrototype · readCodebase│
 │                                    │                                        │
 │  ② CAPTURE         Puppeteer screenshots each state in visualManifest      │
 │                    (drives viewport + clicks; degrades to placeholders)    │
 │                                    │                                        │
 │  ③ INSTRUMENT      one Gainsight data-id plan  ←shared→  analytics + code  │
 │                                    │                                        │
 │  ④ FAN-OUT         warm prompt cache on job #1, then N parallel generates  │
 │                                    │                                        │
 │  ⑤ ARCHIVE         auto-save run (brief + captures + artifacts) to library │
 └──────────────────────────────────────────────────────────────────────────┘
        Review · edit · export · "distribute" all happen in the UI (human in the loop)
```

## The fan-out — 9 audience artifacts + 2 special generators

```
 Change Brief ──┬─► Design System team          ┐
                ├─► Product documentation        │
                ├─► Support & TAMs               │  plain generate()
                ├─► QA                            │  brief + spec, prompt-cached
                ├─► Release notes                 │
                ├─► Analytics & success plan  ◄───┤ (embeds data-id table)
                ├─► 1-Pager  (inline shots)       │
                ├─► Case study (inline shots)     ┘
                ├─► Dev handoff (spec)
                ├─► Slide            → generateObject → SlideSpec → .pptx deck
                ├─► Dev component code → Vue, DS-grounded, data-ids wired in
                └─► + 1 component spec per net-new component (conditional offshoot)
```
