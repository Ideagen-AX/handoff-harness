## The before / after

The manual process this replaces: after every UX change, one designer re-writes the
same change for up to a dozen audiences — release notes, QA cases, product docs, support
summary, DS update, developer handoff, analytics plan, exec 1-pager/slide, case study —
each from scratch, in a different voice, from memory.

| | Before FORGE | After FORGE |
|---|---|---|
| Time spent | **~104 man-hours** to produce a full handoff to the same standard across all audiences | **5 min 25 sec** of runtime (last production run), then review & approve |
| Who touched it | 1 designer, serially, for every artifact | 1 designer reviews; the harness drafts all 9+ outputs in parallel |
| The manual steps | Diff the change by eye · write each artifact from memory · chase house style · capture screenshots by hand · assign analytics ids · stub a starting component | **By design, what remains:** review & edit the Change Brief · review/approve each artifact · send/post/file (manual, human-authorised) |

**In one line — what we stop doing on Monday:** hand-writing the same change a dozen
times over — we write it *once*, as a verifiable change brief, and let the harness voice
it for every audience while a human stays in the loop to approve.
